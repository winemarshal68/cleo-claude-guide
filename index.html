Hey Cleo — you don't need to be technical to get a lot more out of Claude. This is the kit I actually use, arranged easiest → fanciest, with copy-and-paste steps for each.

The one secret worth knowing: you never have to set any of this up alone. Paste any link from this page into Claude and say "walk me through installing this, step by step" — it'll happily do it with you. Start at the top, one thing at a time.

Start here

You've already got the desktop app ✅ — so you're set up. Two spots worth finding inside it: Settings → Connectors (the one-click tool shelf) and Scheduled tasks (we'll use that in Tips). You've also got GitHub and Cloudflare accounts — there are cards for both under Accounts.

The move that makes all of this easy

For anything that looks fiddly, paste the tool's link into Claude and say: "Please install this for me and tell me if you need anything." Claude runs the steps itself — your job is mostly to say yes and grab a login when it asks.

🔑 Plain-English glossary

ConnectorA one-click tool inside the desktop app (Gmail, Notion…). Switch on, sign in, done.
MCPThe plumbing that lets Claude talk to an outside app. You never touch it — you just add the tool.
Skill / PluginA bundle that teaches Claude a new trick. Added with a /plugin command.
API keyA free password a service gives you so Claude can use it. Copy it once, paste it in. That's all.

Keys & 1Password

To let Claude do more in a service you own — save file versions on GitHub, publish a site or edit DNS on Cloudflare — you give it an API key, and 1Password is the safe place to keep those keys. This is the most "grown-up" section; take it slowly, or hand any step to Claude with the 🪄 trick.

🗝️

What's an API key?

Think of it as a spare key-card for a building you already have an office in. Your normal login is you walking in the front door; an API key is a limited card you print for Claude so it can do specific jobs on your behalf. Two things keep it safe: it's scoped (you choose exactly which doors it opens) and revocable (delete it any time and it stops working, your real password untouched).

🐙

Make a GitHub key

Techy

A "fine-grained personal access token" — GitHub's safe, tightly-scoped kind.

  1. Go to github.com/settings/personal-access-tokens → Generate new token (fine-grained).
  2. Name it "Claude", set an expiry, and choose only the repo(s) you want.
  3. Under Permissions, switch Contents and Issues (and Pull requests) to Read & write.
  4. Generate, then copy the github_pat_… — shown once — straight into 1Password.

Make a Cloudflare token

Techy

A scoped Cloudflare API Token — you pick which permissions and which site.

  2. Use a template (e.g. "Edit zone DNS") or a custom one with Pages / DNS edit.
  3. Point it at your specific site, not "All zones".

The golden rule for keys

A key or token is exactly like a password — never paste one into a chat, an email, or anywhere public. It's shown once, so save it into 1Password the moment you see it. Scope it narrowly, give it an expiry, and delete any you've stopped using. If one ever leaks, just delete it and make a new one — instant and free.

Two habits & one rule

Small things that make Claude fit your workday — plus the one line to remember given the information you handle.

⏰

Line up your 5-hour window with your day

Claude refills on a rolling ~5-hour window that starts on your first message — so if you begin at 9am, you might run low mid-afternoon. The fix: have a tiny task run before you arrive, so the refill lands to suit your hours.

9:00 first msg
window runs 9 → 2pm · refills at 2pm 😕 mid-afternoon
7:00 auto-task
quiet start
→ your day
window refills ~noon · fresh for the afternoon 🎉
  • →Desktop app → Scheduled tasks → add a small morning one (even "Good morning, list today's calendar") a couple of hours before you start.
  • →Watch it play out with OpenUsage in your menu bar until the timing feels right.

🎙️ Voice + clipboard = fast

Dictate with FluidVoice, keep your go-to prompts in FlashClip. Together they turn "typing at a chat box" into something closer to talking to a colleague.

📌 Let it remember you

The first thing worth doing: tell Claude who you are and how you like to write (the memory cards above). Everything after lands closer to right the first time.

🛟
One rule, because you work in finance

Be thoughtful with client personal and financial details. Connectors that stay inside your own Gmail, Drive and Calendar are fine — that's your data in your accounts. But don't paste a client's private information into random third-party tools, and keep sensitive files local rather than uploading them somewhere new. Keep logins and keys in 1Password. When in doubt, ask Marshal — better a quick question than a wrong guess.
